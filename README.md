# ai-dtool

`ai-dtool` 是一个面向个人开发者与小团队的本地开发运维工作台，集成了 Redis、Git、Docker、Supervisor、接口调试、任务流程、知识片段和 AI Agent 协作等能力，适合把日常高频的开发与运维操作集中到一个 Web 界面里完成。

## 系统定位

`ai-dtool` 围绕日常开发执行过程设计，重点解决下面几类问题：

- 常用环境和连接信息分散，切换成本高
- Git、Redis、Docker、Supervisor 等操作频繁重复
- 接口调试、接口文档、执行记录分散在不同工具里
- 需求分析、开发执行、测试修复、代码检查缺少统一流程
- 日常知识、日志结果、自动化脚本难以复用和沉淀
- AI Agent 已经能参与开发，但缺少统一的配置、执行和结果查看入口

## 核心功能

### 1. 首页工作台

- 提供统一入口，集中访问常用模块
- 支持快捷命令操作，减少页面切换和重复输入
- 展示异步任务状态，方便查看正在执行、待处理和失败任务
- 适合作为整个系统的日常操作首页

## 适用场景

- 个人开发者管理多个项目、多个环境和多个服务
- 小团队内部沉淀统一的开发操作入口
- 需要频繁进行 Redis、Docker、Git、Supervisor 联动操作
- 希望把接口联调、需求推进、日志排查和知识沉淀集中到一个系统
- 希望将 AI Agent 引入日常开发流程，并保留执行记录和协作上下文
