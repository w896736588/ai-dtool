# 按名称打开智能链接

## 任务描述
用户要求打开某个智能链接（如"打开 link common3"、"用 xxx 账号登录 xxx"），需要：查找智能链接 → 执行打开 → 可选登录。

## 前置条件
- 已知智能链接名称（如 `common3`）
- 智能链接服务正常运行（默认地址 `http://localhost:17170`）

## 执行步骤

### 步骤1: 查询智能链接列表
http_call POST /api/SmartLinkList  
请求体：
```json
{
  "page": 1,
  "page_size": 200
}
```

#### 响应处理
- 关注字段：`Data.smart_link_list` → 智能链接数组
- 遍历数组，查找 `name` 字段等于给定名称 `{tag_name}` 的项，记录其 `id` 作为 `smart_link_id`。
- 若未找到，告知用户该智能链接不存在。

### 步骤2: 执行智能链接
http_call POST /api/SmartLinkRun  
请求体：
```json
{
  "id": {smart_link_id}
}
```
- `{smart_link_id}`：步骤1查到的智能链接 ID

#### 响应处理
- 关注字段：`Data` 中包含执行结果
- 执行成功后告知用户链接已打开
- 若需指定账号登录，执行成功后系统会自动根据链接配置处理登录流程

### 步骤3（可选）: 查询账号信息
当用户指定了账号（如"用1355555..那个账号"），但步骤2的登录未自动匹配时：
http_call POST /api/Set/AccountList  
请求体：
```json
{
  "page": 1,
  "page_size": 100
}
```

#### 响应处理
- 在返回的账号列表中查找匹配用户描述的账号
- 如找到，可补充告知用户找到的账号信息

## 结果汇总
向用户确认：智能链接 `{tag_name}` 已成功打开，并说明是否已自动登录。若未找到匹配链接或账号，需如实告知。
