# 首页任务清单按钮栏设计

## 背景

首页左侧底部当前只有零散按钮，没有统一的底部按钮栏，也缺少全局任务清单能力。需求要求在首页底部增加一排可放置按钮栏，并新增任务清单按钮，通过居中弹窗管理任务。

## 目标

- 首页底部增加统一按钮栏，并保留现有入口。
- 增加“任务清单”按钮，点击后打开居中弹窗。
- 弹窗包含 `活跃中`、`归档` 两个 Tab。
- 支持新增任务。
- 任务状态支持 `待开始`、`进行中`、`暂停`、`已完成`。
- 活跃中默认仅展示 `待开始`、`进行中` 任务。
- 列表展示任务开始时间，格式为年月日。
- 后端记录任务最后操作时间。
- 提供便捷的状态切换接口，支持列表中快速切换状态。

## 范围

### 后端

- 新增首页任务数据表。
- 新增任务查询、保存、归档切换、快捷状态切换接口。
- 在公共数据库访问层封装首页任务读写逻辑。

### 前端

- 首页底部工具区改为统一按钮栏样式。
- 新增任务清单弹窗、Tab、列表、表单与快捷状态操作。
- 新增首页任务接口封装。

## 数据设计

新增表 `tbl_home_task`，字段如下：

- `id`：主键。
- `name`：任务名称。
- `task_status`：任务状态，固定值为 `待开始`、`进行中`、`暂停`、`已完成`。
- `is_archived`：是否归档，`0` 未归档，`1` 已归档。
- `start_time`：任务开始时间，Unix 时间戳，用于显示年月日。
- `last_operated_at`：最后操作时间，Unix 时间戳。
- `create_time`：创建时间。
- `update_time`：更新时间。

说明：

- 当前需求是全局任务清单，因此不引入用户维度字段。
- 只记录最后一次操作时间，不单独建立操作日志表，控制本次改动范围。

## 接口设计

### `POST /api/HomeTaskList`

入参：

- `is_archived`

出参：

- `task_list`

行为：

- `活跃中` 查询 `is_archived = 0`
- `归档` 查询 `is_archived = 1`
- 返回格式化后的开始时间、最后操作时间、更新时间文本

### `POST /api/HomeTaskSave`

入参：

- `id`
- `name`
- `task_status`
- `start_time`

行为：

- `id <= 0` 时新增
- 否则更新
- 新增和更新都会刷新 `last_operated_at` 与 `update_time`

### `POST /api/HomeTaskArchiveToggle`

入参：

- `id`
- `is_archived`

行为：

- 切换归档状态
- 同时刷新 `last_operated_at` 与 `update_time`

### `POST /api/HomeTaskStatusQuickUpdate`

入参：

- `id`
- `task_status`

行为：

- 仅更新任务状态
- 当状态切到 `进行中` 且开始时间为空时，自动写入当前时间作为 `start_time`
- 同时刷新 `last_operated_at` 与 `update_time`

## 前端交互设计

### 首页底部按钮栏

- 现有 `新页卡`、`小工具`、`当前SSH连接数` 保留。
- 新增 `任务清单` 按钮。
- 采用统一按钮栏容器，避免继续堆叠零散标签。

### 任务清单弹窗

- 弹窗居中展示。
- 顶部使用 Tab 切换 `活跃中` 与 `归档`。
- 活跃中默认展示未归档任务中的 `待开始`、`进行中` 项。
- 归档页展示所有已归档任务。
- 列表项显示：
  - 名称
  - 状态
  - 开始时间（年月日）
  - 最后操作时间

### 新增与快捷操作

- 弹窗顶部提供简洁新增表单。
- 每个任务项提供快捷状态按钮，直接切换状态。
- 每个任务项提供归档/取消归档按钮。
- 本次先不做复杂筛选和拖拽排序，保持功能最小闭环。

## 风险与取舍

- 当前活跃列表按需求仅显示 `待开始`、`进行中`，因此未归档但状态为 `暂停`、`已完成` 的任务不会出现在活跃默认列表中。这是按需求实现，不额外扩展筛选器。
- 任务开始时间是否允许手工调整，本次支持在新增/编辑时设置，快捷切换到 `进行中` 时自动补齐空值，避免无开始时间的进行中任务。
