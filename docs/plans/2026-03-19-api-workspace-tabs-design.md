# 接口开发右侧多 Tab 工作区设计

## 背景

当前接口开发页在 [Api.vue](/D:/go/cache_manager_api/web/src/components/Api.vue) 中使用单个 `selectedItem` 驱动右侧内容区。用户每点击一次集合、文件夹或接口，右侧内容都会被新内容覆盖，无法同时保留多个上下文。

这会带来两个问题：

- 用户在多个集合、文件夹、接口之间来回切换时，缺少稳定的工作区。
- 同一节点重复点击时，当前行为是简单覆盖，无法显式表达“切到已有项并重新加载”的需求。

## 目标

- 右侧改为单工作区 tab 池。
- 集合、文件夹、接口都可以在右侧各自占用一个 tab。
- 同一节点只保留一个 tab。
- 重复点击同一节点时，切换到已有 tab，并重新加载该节点数据。
- 用户可手动切换 tab，也可关闭 tab。
- 左侧树当前高亮与右侧激活 tab 保持同步。

## 不做范围

- 本轮不做“刷新页面后恢复右侧 tabs”。
- 不支持同一节点打开多个副本 tab。
- 不重构现有 `CollectionBasicInfo`、`FolderDetail`、`ApiDetail` 的内部表单结构。

## 方案对比

### 方案 A：单工作区 tab 池

右侧维护一个 `openTabs` 数组和一个 `activeTabKey`，集合、文件夹、接口都通过稳定 key 唯一标识。

优点：

- 与“重复点击切到已有 tab 并刷新”的需求完全一致。
- 对现有组件侵入较小，可以保留 `selectedItem` 作为兼容镜像。
- tab 的关闭、刷新、激活规则清晰。

缺点：

- 需要系统性替换当前“只认 `selectedItem`”的更新逻辑。

### 方案 B：单工作区 tab 池 + 手动刷新

重复点击只切换到已有 tab，不自动刷新，由 tab 上的刷新动作手动触发。

优点：

- 网络请求更少。

缺点：

- 不符合本次需求。

### 方案 C：多实例 tab

允许同一节点打开多个 tab 实例。

优点：

- 灵活。

缺点：

- 状态同步、删除联动、树选中联动都会明显复杂。
- 超出当前页面的必要复杂度。

## 选型

采用方案 A：单工作区 tab 池。

## 状态设计

在 [Api.vue](/D:/go/cache_manager_api/web/src/components/Api.vue) 中新增：

- `openTabs`: 当前已打开的工作区 tab 列表。
- `activeTabKey`: 当前激活的 tab key。
- `activeCollectionInnerTabMap`: 记录每个集合 tab 当前内部页签。
- `activeFolderInnerTabMap`: 记录每个文件夹 tab 当前内部页签。

保留现有 `selectedItem`，但其职责变为“当前激活 tab 的数据镜像”，用于兼容现有详情组件和旧逻辑。

### Tab 数据结构

```js
{
  key: 'collection:1',
  type: 'collection',
  id: 1,
  uniqueid: 'collection_1',
  title: '客服系统',
  data: {},
  loaded: false,
  loading: false
}
```

稳定 key 规则：

- `collection:<id>`
- `folder:<id>`
- `api:<id>`

## 交互设计

### 打开

- 首次点击树节点：创建 tab 并切换过去。
- 重复点击同一节点：切换到已有 tab，并重新加载数据。

### 激活

- 点击右侧 tab：切换当前 tab。
- 切换后同步左侧树高亮。
- 切换到接口 tab 时，如其父集合/文件夹尚未展开，应先确保树节点已加载，再定位高亮。

### 关闭

- 用户可关闭集合、文件夹、接口 tab。
- 关闭当前激活 tab 时，切到相邻 tab。
- 关闭最后一个 tab 时，右侧回到空状态。

## 数据刷新策略

### 集合 tab

- 打开或重复点击时，重新拉集合基础信息。
- 若该集合在树上已展开，不清理当前展开状态。

### 文件夹 tab

- 打开或重复点击时，重新拉文件夹基础信息，并刷新该文件夹下接口摘要。
- 确保右侧接口文档和左侧树同步。

### 接口 tab

- 打开或重复点击时，重新拉接口详情。
- 接口详情刷新后，同步 tab 数据和 `ApiDetail` 组件展示。

## 与现有操作的联动

### 创建

- 新建文件夹成功后，自动打开该文件夹 tab 并切换过去。
- 新建接口、复制接口成功后，自动打开新接口 tab 并切换过去。

### 更新

- 集合、文件夹、接口更新成功后，更新对应 tab 的标题和数据。
- 若该 tab 当前激活，则同步到 `selectedItem`。

### 删除

- 删除集合：关闭对应集合 tab。
- 删除文件夹：关闭对应文件夹 tab，并关闭其下所有接口 tab。
- 删除接口：关闭对应接口 tab。

## 视图结构

右侧展示区分为两层：

1. 工作区 tab 条
2. 当前激活 tab 的内容区

内容区继续复用现有组件：

- 集合：`CollectionBasicInfo` + `CollectionEnvironment`
- 文件夹：`FolderDetail`
- 接口：`ApiDetail`

## 风险

- 现有代码中大量逻辑直接依赖 `selectedItem`，需要逐步改成“先更新 tab，再同步当前激活项”。
- `ApiDetail` 通过 ref 初始化，切 tab 时要避免把接口详情误灌到非当前 tab。
- 删除文件夹或接口时，要同时处理树数据、tab 数据、当前激活态。

## 验证要点

- 连续打开多个集合、文件夹、接口，右侧能并存多个 tab。
- 重复点击同一节点时，会切换到已有 tab 并重新加载数据。
- 切换 tab 时，左侧树高亮同步。
- 新建接口后自动打开新接口 tab。
- 删除文件夹或接口后，对应 tab 会被正确关闭。
